# `results/`: model outputs

All numerical outputs and figures. JSON summaries and figures are tracked in
git; large or regenerable binaries are not (see below).

## Naming conventions

| Pattern | Produced by | Contents |
|---|---|---|
| `<model>_tpe_v2_best.json` | cluster Optuna search | Best hyperparameters found by the TPE sampler for that model. |
| `<model>_random_v2_best.json` | cluster Optuna search | Best from the random-search baseline (control). |
| `holdout_<model>.json` | `scripts/07`, `scripts/09` | **Final holdout-test metrics** (AUROC, 95% CI, F1, confusion matrix). The headline numbers. |
| `holdout_<model>_on_<input>.json` | `scripts/08` | One cell of the cross-input experiment (a model evaluated on a non-default representation). |
| `cross_input_summary.json` | `scripts/08` | The full architecture × input grid in one file. |
| `transformer_attention.json` | `scripts/10` | Per-gene `[CLS]` attention ranking for the interpretability analysis. |
| `figures/fig1…fig7_*.png` | `scripts/make_figures.py` (1–6) and `scripts/10` (7) | All report/talk figures, 200 dpi. |

So for any model there are up to three files: the **search** output
(`*_best.json`, what hyperparameters won), and the **final evaluation**
(`holdout_*.json`, how it actually performed on unseen data).

## Tracked vs not tracked

**Tracked (in git):** all `*.json` summaries and all `figures/*.png`. These are
small and are the actual evidence behind the report.

**Not tracked (gitignored, regenerable):**
- `*_predictions.npz`: per-sample `y_true` / `y_prob` arrays. Regenerated by
  re-running the eval scripts; only needed to rebuild figures.
- `optuna_studies.db`: the full SQLite history of every Optuna trial
  (large, machine-specific).
